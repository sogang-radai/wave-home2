## 4. Firmware mapping (r4fn)

| EEPROM part | `FUN_0044bba8` part id | Buffer in `FUN_0041d598` | Application |
|-------------|------------------------|---------------------------|-------------|
| 2 Amplitude | 2 | `local_300` (768 B) | IQ scale `dVar16` per channel |
| 3 Delay | 3 | `local_900` | Multiplier on range index `fIdx ∈ [0,255]` |
| 4 Phase | 4 | `local_600` | Phase offset `fVar19` (rad) in `sincos(fIdx×fVar18+fVar19)` |

Init part 1 is loaded separately (`0x2AC` = **684 B** per cascade device × **4** devices = **2736 B**) into structured mmWave cal (`DAT_0066a8e8` path) — **not** used as the 192-float phasor tables above.

Constants (`FUN_0041dc08`): **12** tiles × **4** RX → **48** rows; **4** phase contexts (columns); **256** points along range cal axis; outer loop **4** passes × **48** channels = **192** indices.

### Index: EEPROM float ↔ cube virtual antenna

Binary layout is **row-major 48×4**: row = `tile×4 + rx` (`tile` 0..11, `rx` 0..3), column = **phase context** 0..3 (firmware outer pass index).

Cube VA index (`docs/CUBE_LAYOUT.md`):

```text
va_index = tile × 16 + sub_ant        # sub_ant ∈ [0,15]
rx       = sub_ant % 4
context  = sub_ant // 4                 # 0..3
row      = tile × 4 + rx              # 0..47
coef_index = row × 4 + context
         = row + 48 × context
         = tile×4 + (sub_ant%4) + 48×(sub_ant//4)
```

Same index orders the **192 floats** in each of `ampli_coefs.bin`, `delay_coefs.bin`, `phase_coefs.bin`.

**Text on sensor:** `srs_eeprom` labels parts 2–4 as `*.txt`; factory `change_eeprom_cal_data.sh` / `autocheck` writes **text** from `cfg_profile`. Exported `*.bin` are **raw float32 LE** — not the ASCII row format.

## 9. Init calibration (`initcal.bin`)

| Property | Value |
|----------|-------|
| Size | 2736 B = **4 × 684 B** |
| Cascade map | Profile uses **4** radar devices (`CascadeDeviceMap=15`) |
| Content | Structured **mmWaveLink init cal** records (mixed `uint16` / packed fields), **not** a `float[684]` array |
| Human view | `initcal.load.txt` = `srs_eeprom load 1` decode (hex + parsed fields) |
| Firmware | `FUN_0044bba8(..., part=1, size=0xAB0)` loads into `DAT_0066a8e8`; per-device stride **0x2AC (684)** |

Do **not** feed `initcal.bin` into the same 192-VA phasor path as phase/ampli/delay. It configures TI chirp/RX/TX factory calibration at boot.

## 11. Offline software use (vital signs)

For one range bin `r` and VA index `va = tile×16 + sub_ant`:

```python
row = tile * 4 + (sub_ant % 4)
col = sub_ant // 4
i = row * 4 + col
phi = phase[i]          # radians
amp = ampli[i]
# range-dependent correction (firmware uses fIdx=0..255):
# angle(r) = delay[i] * r + phi  (then exp(j*angle) * amp)
```

Cube from `r4sn_cube_log` is **after** much of this chain; use these coefs when re-steering or comparing to simulation.
