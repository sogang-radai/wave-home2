# Style
이 문서는 이 워크스페이스에서 준수해야 할 코드 스타일을 정의합니다.

## C++ Style
헤더
- 헤더파일은 STL, 외부, 프로젝트 헤더 순서로 작성(프로젝트 헤더는 한 줄 띄움).
- 소스파일에 대응하는 헤더파일은 항상 소스파일의 첫 번째 헤더로 작성(그 아래에 한 줄 띄움).
- 프로젝트 헤더는 common/을 참조하는 경우를 포함해서 ""로 시작하도록 작성.
- 프로젝트 헤더에서 common/을 참조하는 경우가 아니라면 상대경로를 사용.

네임스페이스
- 사용자 정의 네임스페이스의 경우 프로젝트에 ???defs.h와 같은 헤더를 만들고, ??_NAMESPACE_BEGIN, ??_NAMESPACE_END 매크로를 정의하여 사용.
- 사용자 정의 네임스페이스를 열고 닫는 경우 방금 만든 매크로를 위아래로 사용.
- 네임스페이스 영역은 매크로를 사용하는 경우를 제외하고는 항상 들여쓰기 수행.
- 익명 네임스페이스의 경우 항상 들여쓰기 수행.

타입
- 가능하면 size_t, ptrdiff_t, int32_t, uint32_t, int64_t, uint64_t 등과 같은 고정 크기 타입을 사용.
- 위 타입을 사용할 때 std::는 생략

변수
- 지역 변수는 항상 소문자로 시작하는 snake case를 사용.
- 함수 인자는 짧고 간단 명료하게 작성하며, 소문자로 시작하는 snake case를 사용.
- 전역 변수는 g_ 접두사를 사용하며, 소문자로 시작하는 camel case를 사용(static 변수는 s_ 접두사를 사용).
- 클래스나 구조체의 private 멤버는 m_ 접두사를 사용하며 소문자로 시작하는 camel case 사용
- 클래스나 구조체의 public 멤버는 소문자로 시작하는 camel case 사용

제어문
- 아래 예시 처럼 가능하면 중괄호를 늘려서 쓰는 스타일을 사용.
- 다만 구문이 한 줄로 끝나는 경우에는 중괄호를 생략 가능.
- if-else에서 하나만 구문이 한 줄로 끝나는 경우에는 해당 구문만 중괄호를 생략 가능.
``` cpp
if (condition)
{
    ...
}
else
{
    ...
}
```

함수
- 다른 소스에서 참조될 수 있는 함수는 소문자로 시작하는 camel case를 사용.
- 익명 네임스페이스나 static 함수의 경우 소문자로 시작하는 snake case를 사용.
- 클래스 멤버는 static이 아닌 경우 항상 소문자로 시작하는 camel case를 사용.
- 클래스 멤버는 static인 경우 항상 소문자로 시작하는 snake case를 사용.

생성자
- 생성자에서 멤버 초기화는 멤버 초기화 리스트를 사용.
- 아래 스타일을 기본으로 하되, 만약 본문 내용이 없다면 m_b(b) {} 와 같이 중괄호를 한 줄에 작성 가능.
``` cpp
MyClass::MyClass(int a, int b) :
    m_a(a),
    m_b(b)
{
    ...
}
```
